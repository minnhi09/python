### Khái niệm cơ bản

Thụt lề (indentation) là một yếu tố **vô cùng quan trọng** trong Python. Khi tạo một hàm (function), mọi dòng code được thụt lề (dịch sang phải 4 khoảng trắng) sau định nghĩa hàm sẽ nằm **bên trong** hàm đó.

```python
def my_function():
    print("Hello")    # Nằm trong hàm
    print("World")    # Nằm trong hàm
```

Nếu một dòng code **không được thụt lề**, nó sẽ độc lập với hàm và không được thực thi khi hàm được gọi:

```python
def my_function():
    print("Hello")    # Nằm trong hàm
print("World")        # Nằm ngoài hàm
```


### Cách hình dung thụt lề

Bạn có thể hình dung thụt lề giống như **cấu trúc thư mục** (folder structure) trên máy tính:

- Hàm giống như một **thư mục**
- Các dòng code bên trong hàm giống như **các file trong thư mục**
- Khi một file nằm trong thư mục, nó được thụt lề vào bên trong


### Thụt lề lồng nhau

Khi có các khối lệnh (blocks) lồng nhau như `if/elif/else` hoặc `for loops`, mỗi khối cần thêm 4 khoảng trắng:

```python
def my_function():
    # Khối function - thụt lề 4 spaces
    if sky == "clear":
        # Khối if - thụt lề thêm 4 spaces (tổng 8)
        print("blue")
    elif sky == "cloudy":
        # Khối elif - thụt lề thêm 4 spaces
        print("grey")
```


### Spaces vs Tabs - Cuộc tranh luận vĩnh cửu

Có hai cách tạo thụt lề trong Python:

- **Spaces (khoảng trắng)**: Nhấn phím Space 4 lần
- **Tabs (tab)**: Nhấn phím Tab 1 lần

Theo **hướng dẫn chính thức** từ cộng đồng Python (PEP 8 Style Guide):

- **Spaces là phương pháp được khuyến nghị**
- Mỗi mức thụt lề = **4 khoảng trắng**
- **Không được trộn lẫn** spaces và tabs trong cùng một file (Python 3 sẽ báo lỗi)


### Nghiên cứu thú vị

Theo khảo sát StackOverflow Developer Survey năm 2017, các lập trình viên sử dụng spaces có mức lương trung bình **cao hơn** những người sử dụng tabs.

### Mẹo thực hành

Hầu hết các trình soạn thảo code (code editors) hiện đại có tính năng:

- Chuyển đổi phím Tab thành 4 spaces tự động
- Bạn chỉ cần nhấn Tab **một lần**, editor sẽ chèn 4 spaces
- Điều này giúp code tuân thủ chuẩn Python mà vẫn tiện lợi khi gõ


### Ghi chú quan trọng

- Luôn chú ý đến thụt lề khi viết code Python
- Hãy "nhìn thấy" các khối code (blocks) thông qua mức độ thụt lề
- Tham khảo PEP 8 Style Guide để nắm rõ các quy tắc chuẩn
