### Cú pháp tạo Class

**Cấu trúc cơ bản:**

```python
class ClassName:
    # Nội dung class (indented)
```

**Thành phần:**

- Keyword `class` - Từ khóa khai báo class
- Class name - Tên class (PascalCase)
- Dấu hai chấm `:` - Kết thúc khai báo
- Indentation - Nội dung class phải thụt vào


### Ví dụ thực tế: User Class

**Scenario:** Xây dựng website cần mô hình hóa users

**Tạo class User:**

```python
class User:
    pass
```

**Giải thích:**

- `User` = Class blueprint đại diện cho website users
- `pass` = Keyword để tạo class rỗng tạm thời

**Tạo object từ class:**

```python
# Tạo user object đầu tiên

```

**Cách hoạt động:**

- `user_1` = Tên object (snake_case)
- `User()` = Gọi class với dấu ngoặc đơn để khởi tạo
- PyCharm hiển thị ký hiệu `C` bên cạnh tên class


### Keyword 'pass'

**Vấn đề:**
Python không cho phép tạo class hoặc function rỗng mà không có nội dung.

```python
# Lỗi - Indent Expected
class User:

print("hello")
```

**Giải pháp - Dùng 'pass':**

```python
class User:
    pass  # Tạm thời để trống, sẽ thêm code sau

print("hello")  # OK - Không còn lỗi
```

**Công dụng của 'pass':**

- Placeholder (chỗ giữ chỗ) cho code sẽ viết sau
- Báo cho Python: "Tôi muốn để trống phần này tạm thời"
- Cho phép code chạy mà không bị lỗi syntax

**Áp dụng tương tự cho functions:**

```python
def my_function():
    pass  # Sẽ implement sau
```


### Quy tắc Đặt tên trong Python

**PascalCase - Cho Class names:**

```python
class User:           # ✓ Đúng
class MyClass:        # ✓ Đúng  
class UserProfile:    # ✓ Đúng
class WebsiteUser:    # ✓ Đúng
```

**Đặc điểm PascalCase:**

- Chữ cái đầu của MỖI từ viết hoa
- Không có dấu gạch dưới
- Không có khoảng trắng
- Giống cách viết tên người: Blaise Pascal

**camelCase - Ít dùng trong Python:**

```python
myVariable = 10       # Ít gặp trong Python
userName = "John"     # Không phổ biến
```

**Đặc điểm camelCase:**

- Từ đầu tiên viết thường
- Các từ sau viết hoa chữ cái đầu
- Phổ biến trong JavaScript, Java

**snake_case - Cho hầu hết mọi thứ khác:**

```python
user_1 = User()              # Variables
def calculate_total():       # Functions
my_coffee_maker = CoffeeMaker()  # Objects
MAX_SPEED = 100              # Constants (UPPER_SNAKE_CASE)
```

**Đặc điểm snake_case:**

- Tất cả chữ thường
- Ngăn cách bằng dấu gạch dưới `_`
- Dễ đọc, rõ ràng


### Tóm tắt Naming Conventions

**Trong Python:**


| Loại | Style | Ví dụ |
| :-- | :-- | :-- |
| Class names | PascalCase | `User`, `CoffeeMaker` |
| Variables | snake_case | `user_name`, `total_price` |
| Functions | snake_case | `calculate_sum()`, `get_data()` |
| Objects | snake_case | `my_user`, `coffee_maker` |
| Constants | UPPER_SNAKE_CASE | `MAX_SIZE`, `PI` |

**Không dùng trong Python:**

- camelCase - hiếm gặp

### Ý nghĩa của Class Blueprint

**Class = Blueprint (Bản thiết kế):**

- Định nghĩa cấu trúc chung
- Quy định attributes và methods
- Có thể tạo nhiều objects từ một class

**Ví dụ thực tế:**

```python
class User:
    pass

# Tạo nhiều users từ cùng blueprint
user_1 = User()  # User thứ nhất
user_2 = User()  # User thứ hai  
user_3 = User()  # User thứ ba

# Mỗi user là object riêng biệt
print(user_1)  # <__main__.User object at 0x1234>
print(user_2)  # <__main__.User object at 0x5678>
```


### Bước tiếp theo

Class hiện tại còn rỗng (`pass`). Trong các bài sau sẽ học:

- Thêm attributes vào class
- Thêm methods vào class
- Khởi tạo với `__init__()`
- Sử dụng keyword `self`
