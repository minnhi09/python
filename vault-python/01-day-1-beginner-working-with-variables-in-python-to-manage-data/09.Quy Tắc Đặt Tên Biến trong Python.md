


### Nguyên Tắc Cơ Bản

Bạn có thể đặt tên biến tùy ý, nhưng cần tuân thủ một số quy tắc quan trọng để code dễ đọc và tránh lỗi.

**Ví dụ tên biến hợp lệ:**

```python
name = "Angela"
n = "Angela"
username = "Angela"
```

Miễn là bạn **nhất quán** khi sử dụng tên biến, code sẽ hoạt động:

```python
n = input("What is your name? ")
l = len(n)
print(l)
```


### Nguyên Tắc 1: Code Phải Dễ Đọc (Readability)

**Tránh tên biến quá ngắn hoặc mơ hồ:**

```python
# ❌ Không nên - khó hiểu sau một thời gian
n = input("What is your name? ")
l = len(n)

# ✅ Nên dùng - rõ ràng, dễ hiểu
name = input("What is your name? ")
length = len(name)
```

**Lý do:** Khi quay lại code sau 6-12 tháng, tên biến có ý nghĩa sẽ giúp bạn hiểu nhanh hơn.

### Nguyên Tắc 2: Sử Dụng Underscore (_) để Tách Từ

Trong Python, sử dụng **snake_case** (gạch dưới) để tách các từ trong tên biến.

**Đúng:**

```python
username = "Angela"
user_name = "Angela"
first_name = "Angela"
```

**Sai:**

```python
user name = "Angela"  # ❌ SyntaxError - không được có khoảng trắng
```

**Giải thích:** Tên biến phải là một đơn vị duy nhất, không thể có khoảng trắng. Dùng `_` để thay thế khoảng trắng.

### Nguyên Tắc 3: Sử Dụng Số trong Tên Biến

**Quy tắc:**

- Số có thể xuất hiện trong tên biến
- Số **không được** ở đầu tên biến

**Đúng:**

```python
length1 = 5
length2 = 10
name3 = "Jack"
```

**Sai:**

```python
1length = 5   # ❌ SyntaxError
3name = "Jack" # ❌ SyntaxError
```


### Nguyên Tắc 4: Không Dùng Tên Hàm Làm Tên Biến

**Tránh dùng tên các hàm tích hợp (built-in functions) làm tên biến:**

```python
# ❌ Không nên - gây nhầm lẫn
print = "Hello"
input = "Something"
len = 10
```

**Lý do:**

- Syntax highlighting bị lộn xộn
- Gây nhầm lẫn giữa biến và hàm
- Có thể gây lỗi khó phát hiện
- Vi phạm best practice

**Cách nhận biết:** Nếu tên biến có cùng màu highlighting với các hàm, hãy đổi tên.

### NameError - Lỗi Tên Biến

**NameError** xảy ra khi bạn sử dụng tên biến chưa được định nghĩa hoặc viết sai chính tả.

**Ví dụ lỗi:**

```python
name = "Angela"
print(nama)  # ❌ NameError: name 'nama' is not defined
```

**Nguyên nhân:** `name` (đúng) khác với `nama` (sai chính tả).

**Cách khắc phục:**

1. Đọc thông báo lỗi để biết dòng code bị lỗi
2. Kiểm tra chính tả tên biến
3. Đảm bảo tên biến được viết giống hệt lúc định nghĩa

### Tính Nhất Quán (Consistency) là Chìa Khóa

Python **không** kiểm tra chính tả - nó chỉ quan tâm đến tính nhất quán.

**Ví dụ hợp lệ (dù sai chính tả):**

```python
nama = "Angela"  # Định nghĩa biến (có gạch dưới xanh lá từ spell checker)
print(nama)      # Sử dụng biến - Hoạt động bình thường!
```

**Giải thích:**

- Python không biết "nama" có phải lỗi chính tả hay không
- Miễn là bạn nhất quán: định nghĩa `nama` và dùng `nama` → Không lỗi
- Có thể bạn đang tạo nhân vật game tên "Nama" → Hoàn toàn hợp lệ

**Quy tắc vàng:** Tên biến khi **định nghĩa** và khi **sử dụng** phải **giống hệt nhau**.

### Bảng Tổng Hợp Quy Tắc

| Quy tắc | Hợp lệ ✅ | Không hợp lệ ❌ |
| :-- | :-- | :-- |
| Tách từ | `user_name` | `user name` |
| Số trong tên | `name1`, `length2` | `1name`, `3length` |
| Tên hàm | `my_print`, `user_input` | `print`, `input`, `len` |
| Chính tả | `name` → `name` | `name` → `nama` |
| Ký tự đặc biệt | `user_name`, `name123` | `user-name`, `user@name` |

### Mẹo Nhận Biết Lỗi Nhanh

**Khi gặp NameError:**

1. Kiểm tra chính tả tên biến
2. So sánh tên biến lúc định nghĩa và lúc sử dụng
3. Kiểm tra có viết hoa/thường đúng không (Python phân biệt: `Name` ≠ `name`)

**Khi thấy syntax highlighting lạ:**

- Tên biến có màu giống hàm → Đổi tên
- Tên biến bị gạch chân đỏ → Kiểm tra cú pháp
- Tên biến bị gạch chân xanh → Lỗi chính tả (warning, không phải lỗi code)


### Thực Hành Tốt (Best Practices)

**Nên làm:**

```python
# Tên mô tả rõ ràng
user_age = 25
total_price = 100.5
first_name = "John"

# Nhất quán trong cả chương trình
username = input("Enter name: ")
print(f"Hello {username}")
```

**Không nên:**

```python
# Tên quá ngắn, khó hiểu
a = 25
x = 100.5

# Dùng tên hàm
input = "some text"
print = "Hello"
```


### Ghi Chú Thêm

- Python **phân biệt chữ hoa/thường**: `Name`, `name`, `NAME` là ba biến khác nhau
- Gạch dưới xanh lá từ spell checker không ảnh hưởng code
- Tính nhất quán quan trọng hơn chính tả đúng
- Đặt tên biến tốt = code tự giải thích (self-documenting code)
