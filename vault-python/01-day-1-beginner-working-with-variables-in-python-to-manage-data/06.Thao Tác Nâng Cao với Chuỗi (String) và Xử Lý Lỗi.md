### Ký tự Xuống Dòng (\n - New Line)

Thay vì viết nhiều câu lệnh `print()` riêng biệt, bạn có thể sử dụng ký tự đặc biệt `\n` để xuống dòng trong một câu lệnh duy nhất.

**Ví dụ:**

```python
# Cách cũ - nhiều lệnh print
print("Hello world!")
print("Hello world!")
print("Hello world!")

# Cách mới - một lệnh print với \n
print("Hello world!\nHello world!\nHello world!")
```

**Lưu ý:** Sử dụng dấu gạch chéo ngược `\` (backslash), không phải gạch chéo `/` (forward slash). Syntax highlighting sẽ giúp bạn nhận biết đúng cú pháp.

### String Concatenation (Nối chuỗi)

**Concatenation** là kỹ thuật kết hợp nhiều chuỗi ký tự thành một chuỗi duy nhất bằng cách sử dụng dấu cộng `+`.

**Ví dụ cơ bản:**

```python
print("Hello" + "Angela")  # Kết quả: HelloAngela
```

**Thêm khoảng trắng:**

Có ba cách để thêm khoảng trắng giữa các chuỗi:

```python
print("Hello " + "Angela")      # Thêm space vào cuối "Hello"
print("Hello" + " Angela")      # Thêm space vào đầu "Angela"
print("Hello" + " " + "Angela") # Thêm space như một chuỗi riêng
```

Cả ba cách đều cho kết quả: `Hello Angela`

### IndentationError (Lỗi thụt lề)

**IndentationError** xảy ra khi có khoảng trắng hoặc thụt lề không mong muốn ở đầu dòng code.

**Ví dụ code bị lỗi:**

```python
 print("Hello" + " " + "Angela")  # Có space ở đầu dòng → Lỗi!
```

**Nguyên nhân phổ biến:**

- Thêm khoảng trắng (space) ở đầu dòng
- Nhấn phím Tab ở đầu dòng một cách vô tình

**Cách khắc phục:** Xóa tất cả khoảng trắng ở đầu dòng code. Trong Python, khoảng trắng trong code rất quan trọng và có ý nghĩa cú pháp.

### Phân Biệt Các Loại Lỗi

| Loại lỗi | Nguyên nhân | Ví dụ |
| :-- | :-- | :-- |
| **SyntaxError** | Thiếu dấu ngoặc kép, dấu ngoặc đơn, sai cú pháp | `print("Hello)` |
| **IndentationError** | Thụt lề hoặc khoảng trắng không đúng ở đầu dòng | ` print("Hello")` |

### Cảnh Báo PEP 8 (Warnings)

PyCharm có thể hiển thị cảnh báo màu vàng về phong cách code, ngay cả khi code vẫn chạy được.

**Ví dụ:**

```python
print ("Hello")  # Cảnh báo: không nên có space trước dấu ngoặc đơn
```

**Theo PEP 8:** Không nên có khoảng trắng (whitespace) trước dấu ngoặc mở.

### Khoảng Trắng Trong Python

**Quy tắc quan trọng:**

- **Khoảng trắng bên trong chuỗi** (giữa dấu ngoặc kép): An toàn, là một phần của string
- **Khoảng trắng trong code** (bên ngoài chuỗi): Rất quan trọng, có thể gây lỗi

**Nhận biết:** Phần code màu xanh lá (string) có thể chứa khoảng trắng tự do. Phần code màu khác cần cẩn thận với khoảng trắng.

### Debugging - Khái niệm và Nguồn gốc

**Debugging** là quá trình tìm và sửa lỗi trong code.

**Nguồn gốc thuật ngữ:** Vào những năm 1980, một con bướm đêm (moth) bay vào máy tính đời đầu và bị điện giật. Điều này khiến code không chạy đúng, và lập trình viên phải "gỡ con bướm" (de-bug) ra khỏi máy tính để sửa lỗi.

**Ngày nay:** Chúng ta không có bướm đêm trong code, nhưng vẫn phải "gỡ lỗi" để code chạy đúng như mong đợi.

### Lời Khuyên Về Editor (Trình soạn thảo)

**PyCharm cung cấp:**

- Gạch chân lỗi (squiggly lines)
- Gợi ý khi rê chuột
- Cảnh báo về phong cách code

**Lưu ý quan trọng:** Các editor khác có thể không có các tính năng này. Vì vậy, hãy tập trung vào việc đọc và hiểu thông báo lỗi xuất hiện sau khi chạy code (từ Python compiler), vì những thông báo này giống nhau ở mọi môi trường.

### Thực Hành

Hãy thử nghiệm:

- Sử dụng `\n` để tạo nhiều dòng text
- Nối các chuỗi với dấu `+`
- Cố tình tạo lỗi để quan sát thông báo lỗi
- Sửa các lỗi dựa trên thông báo từ Python
