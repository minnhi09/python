


### Khái niệm hàm Print (In dữ liệu)

Hàm **print()** là lệnh cơ bản giúp hiển thị thông tin ra console (vùng xuất kết quả). Cú pháp cơ bản:

```python
print("Hello world!")
```

**Cấu trúc:**

- Từ khóa `print` viết thường
- Dấu ngoặc đơn `()`
- Nội dung cần in đặt bên trong ngoặc đơn


### Hiểu về String (Chuỗi ký tự)

**String** là chuỗi các ký tự văn bản, được đặt trong dấu ngoặc kép `" "` để phân biệt với code. Dấu ngoặc kép đánh dấu điểm bắt đầu và kết thúc của chuỗi ký tự.

```python
print("Hello world!")  # "Hello world!" là một string
```

**Tại sao cần dấu ngoặc kép?**

- Giúp máy tính phân biệt văn bản thường với từ khóa lập trình
- Nếu thiếu dấu ngoặc kép đóng, code sẽ báo lỗi SyntaxError


### Vùng Console/Output (Khu vực xuất kết quả)

Khi chạy code, vùng console hiển thị 3 phần:

- **Dòng đầu:** Đường dẫn file đang chạy (vị trí file code)
- **Dòng giữa:** Kết quả xuất ra từ lệnh của bạn
- **Dòng cuối:** Thông báo trạng thái (ví dụ: `Process finished with exit code 0` nghĩa là chạy thành công không lỗi)


### Lỗi SyntaxError - Lỗi Cú Pháp

**Nguyên nhân phổ biến:**

- Thiếu dấu ngoặc kép đóng
- Thiếu dấu ngoặc đơn đóng
- Sai cú pháp code

**Ví dụ code bị lỗi:**

```python
print("Hello world!)  # Thiếu dấu ngoặc kép đóng
```

**Cách xử lý lỗi:**

1. Đọc thông báo lỗi màu đỏ trong console
2. Copy toàn bộ thông báo lỗi
3. Tìm kiếm trên Google hoặc [[StackOverflow]]
4. Áp dụng giải pháp từ cộng đồng

### Syntax Highlighting (Làm nổi bật cú pháp)

PyCharm tự động tô màu code để giúp phát hiện lỗi:

- **Màu trắng:** Dấu ngoặc đơn, ký hiệu code
- **Màu xanh lá:** Chuỗi văn bản (string)
- Nếu màu sắc bất thường → Có thể có lỗi cú pháp

**Dấu hiệu lỗi trong PyCharm:**

- **Gạch chân màu đỏ:** Lỗi nghiêm trọng, code không chạy được
- **Gạch chân màu vàng:** Cảnh báo, code vẫn chạy nhưng không tuân thủ chuẩn


### PEP 8 - Tiêu Chuẩn Viết Code Python

**PEP 8** là bộ hướng dẫn về phong cách viết code Python chuẩn. Một quy tắc cơ bản: Nên có dòng trống ở cuối file code.

**Ví dụ:**

```python
print("Hello world!")
# ← Nên có dòng trống ở đây
```


### Kỹ năng Debug (Gỡ lỗi)

**Quy trình debug hiệu quả:**

- Quan sát màu sắc code (syntax highlighting)
- Di chuột qua gạch chân màu đỏ/vàng để xem gợi ý
- Đọc thông báo lỗi và dấu mũ `^` chỉ vị trí lỗi
- Tìm kiếm lỗi trên Google/StackOverflow
- Hiểu nguyên nhân trước khi sửa (không sửa mò)

**Lưu ý quan trọng:** Mọi lập trình viên đều mắc lỗi. Điểm khác biệt của lập trình viên giỏi là khả năng tìm và sửa lỗi nhanh chóng.

### Phím Tắt Hữu Ích

    - Windows/Linux: Xem gợi ý khi rê chuột vào nút Play
