

## Import Modules Trong Python - Các Phương Pháp

### Import Cơ Bản (Basic Import)

Cách đơn giản nhất để import module:

```python
import turtle

# Sử dụng class từ module
my_turtle = turtle.Turtle()
```

**Cú pháp:** `import [Tên Module]`

**Cách sử dụng:** Phải viết tên module trước tên class/function: `module_name.ClassName()`

### From Import - Import Từ Module

Khi sử dụng một class/function nhiều lần, cách này tiện lợi hơn:

```python
from turtle import Turtle

# Tạo nhiều turtle không cần viết turtle.Turtle()
tim = Turtle()
tom = Turtle()
terry = Turtle()
```

**Cú pháp:** `from [Tên Module] import [Thing in Module]`

**Ưu điểm:** Không cần lặp lại tên module nhiều lần, code ngắn gọn hơn.

**Khi nào nên dùng:** Khi sử dụng một thành phần từ module từ 3 lần trở lên.

### Import Everything (*) - Import Tất Cả

Cách import này cho phép sử dụng mọi thứ trong module:

```python
from turtle import *
from random import *

# Sử dụng trực tiếp không cần tên module
forward(100)
choice([1, 2, 3])
```

**Cú pháp:** `from [Tên Module] import *`

**⚠️ Nhược điểm nghiêm trọng:**

- Rất khó biết method/class đến từ module nào
- Code khó đọc và khó bảo trì
- Không rõ nguồn gốc của các function

**Khuyến nghị:** Tránh sử dụng cách này trong code thực tế. Đây là practice không tốt trong cộng đồng Python.

### Import Với Alias - Đặt Tên Rút Gọn

Tạo tên thay thế ngắn gọn cho module:

```python
import turtle as t

# Sử dụng alias thay vì tên module đầy đủ
my_turtle = t.Turtle()
```

**Cú pháp:** `import [Tên Module] as [Alias Name]`

**Khi nào nên dùng:** Khi tên module quá dài và cần viết nhiều lần.

### Python Standard Library vs External Packages

**Standard Library (Thư viện chuẩn):**

```python
import turtle  # Có sẵn, không cần cài đặt
```

Standard Library là tập hợp các module cơ bản đi kèm Python, giống như "thư viện gia đình" nhỏ với những thứ thiết yếu.

**External Packages (Gói bên ngoài):**

```python
import heroes  # Cần cài đặt trước
```

Khi gặp lỗi `No module named 'heroes'`, có nghĩa module này không có trong Standard Library và cần cài đặt.

**Cách cài đặt trong PyCharm:**

- Click vào lỗi có gạch chân đỏ
- Click bóng đèn đỏ xuất hiện
- Chọn "Install package heroes"


### Virtual Environment (Môi Trường Ảo)

**Khái niệm:**

Virtual Environment (venv) là "sandbox" (môi trường cách ly) riêng cho mỗi project Python.

**Tại sao cần Virtual Environment:**

```
Project A (Python 2) → venv A → heroes v1.0
Project B (Python 3) → venv B → heroes v2.0
```

**Lý do lịch sử:**

- Python 2 và Python 3 không tương thích ngược (not backward-compatible)
- Code Python 2 không chạy được với Python 3
- Virtual environment giúp cách ly các phiên bản

**Lợi ích của Virtual Environment:**

- Mỗi project có môi trường riêng biệt
- Cài đặt modules tương thích với phiên bản Python của project
- "Đóng băng" (freeze) project với các dependencies cụ thể
- Đảm bảo project chạy đúng ngay cả khi Python hoặc modules cập nhật phiên bản mới

**Vị trí lưu trữ:**

Trong PyCharm, virtual environment được lưu trong thư mục `.venv` của project:

```
day-18-start/
├── .venv/
│   ├── heroes/
│   ├── villains/
│   └── ...
├── main.py
└── ...
```

**Lưu ý quan trọng:** Không cần chỉnh sửa thư mục `.venv` thủ công, PyCharm tự động quản lý.

### So Sánh: Software Install vs Package Install

**Software (Phần mềm):**

- Cài đặt một lần, sử dụng toàn bộ máy tính (globally)
- Ví dụ: PyCharm, Chrome, VS Code

**Python Packages:**

- Cài đặt riêng cho từng project (locally)
- Lưu trong virtual environment của project đó
- Project mới không tự động có packages cũ


### Cài Đặt Packages - Best Practices

**Phương pháp khuyến nghị: Dùng PyCharm**

Ưu điểm:

- Tự động cài vào đúng virtual environment
- Tránh lỗi cài sai vị trí
- Đảm bảo tương thích với các packages khác
- Dễ quản lý phiên bản

**Phương pháp không khuyến nghị: pip trong terminal**

```bash
pip install heroes  # Có thể cài sai vị trí
```

Vấn đề: Dễ cài vào sai virtual environment, dẫn đến project không truy cập được module.

### Tóm Tắt Các Phương Pháp Import

**1. Import cơ bản:** Dùng khi chỉ sử dụng 1-2 lần

```python
import turtle
turtle.Turtle()
```

**2. From import:** Dùng khi sử dụng từ 3 lần trở lên

```python
from turtle import Turtle
Turtle()
```

**3. Import everything (*):** Tránh sử dụng trong code thực tế

```python
from turtle import *  # ❌ Không khuyến nghị
```

**4. Import với alias:** Dùng khi tên module dài

```python
import turtle as t
t.Turtle()
```


### Nguyên Tắc Code Tốt

**Code rõ ràng:**

```python
import random
random.choice([1, 2, 3])  # ✅ Rõ ràng method đến từ đâu
```

**Code không rõ ràng:**

```python
from random import *
choice([1, 2, 3])  # ❌ Khó biết choice() từ module nào
```

**Mục tiêu:** Code phải "expressive" (diễn đạt rõ ràng), người đọc dễ hiểu nguồn gốc của các method và class.
