

## Thử Thách 1: Vẽ Hình Vuông Với Turtle

### Mô Tả Bài Tập

Thử thách đầu tiên là bài tập khởi động đơn giản: sử dụng turtle để vẽ một hình vuông 100x100. Không quan trọng vị trí trên màn hình, chỉ cần vẽ được hình vuông là thành công.

### Giải Pháp 1: Code Lặp Lại

Cách tiếp cận trực quan nhất:

```python
from turtle import Turtle, Screen

timmy_the_turtle = Turtle()

# Vẽ 4 cạnh của hình vuông
timmy_the_turtle.forward(100)
timmy_the_turtle.left(90)
timmy_the_turtle.forward(100)
timmy_the_turtle.left(90)
timmy_the_turtle.forward(100)
timmy_the_turtle.left(90)
timmy_the_turtle.forward(100)
timmy_the_turtle.left(90)

screen = Screen()
screen.exitonclick()
```

**Logic:** Di chuyển tiến 100 bước, sau đó xoay trái (hoặc phải) 90 độ. Lặp lại 4 lần cho 4 cạnh của hình vuông.

### Giải Pháp 2: Sử Dụng Vòng Lặp For (Tối Ưu)

Vì lập trình viên không thích code lặp lại, sử dụng vòng lặp là giải pháp tốt hơn:

```python
from turtle import Turtle, Screen

timmy_the_turtle = Turtle()

# Vẽ hình vuông bằng vòng lặp
for _ in range(4):
    timmy_the_turtle.forward(100)
    timmy_the_turtle.left(90)

screen = Screen()
screen.exitonclick()
```

**Ưu điểm:**

- Chỉ cần 3 dòng code thay vì nhiều dòng lặp lại
- Code sạch, dễ đọc và dễ bảo trì
- Sử dụng `range(4)` để lặp 4 lần tương ứng với 4 cạnh


### Kỹ Thuật Refactor Trong PyCharm

Khi tên biến quá dài (như `timmy_the_turtle`), việc thay đổi thủ công ở nhiều chỗ rất mất thời gian. PyCharm cung cấp tính năng Refactor (tái cấu trúc) để đổi tên nhanh chóng:

**Cách thực hiện:**

- Click phải vào tên biến
- Chọn "Refactor" → "Rename"
- Nhập tên mới (ví dụ: `tim`)
- PyCharm tự động thay đổi tất cả các vị trí sử dụng biến đó

```python
# Sau khi refactor
tim = Turtle()

for _ in range(4):
    tim.forward(100)
    tim.left(90)
```


### Điểm Cần Lưu Ý

**Về logic vẽ hình vuông:**

- Mỗi góc của hình vuông là 90 độ
- Cần di chuyển và xoay tổng cộng 4 lần
- Có thể dùng `left(90)` hoặc `right(90)` đều được

**Về coding style:**

- Tránh lặp code không cần thiết
- Sử dụng vòng lặp khi có pattern lặp lại
- Đặt tên biến ngắn gọn nhưng có ý nghĩa


### Nguyên Tắc Làm Bài

Đây là bài tập khởi động (warm-up), cả hai giải pháp đều được chấp nhận. Tuy nhiên, giải pháp sử dụng vòng lặp for thể hiện tư duy lập trình tốt hơn.
