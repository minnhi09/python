

## Tại sao OOP hữu ích - Ví dụ từ Steve Jobs

### Câu chuyện khách sạn ở Nhật Bản

**Tình huống:**

Bạn đến Nhật Bản sau một chuyến bay dài. Chiếc áo phông của bạn bị bẩn do đổ nước cà chua. Bạn gặp nhiều vấn đề:

- Không biết vị trí hiện tại
- Không biết tiệm giặt ủi ở đâu
- Không có tiền tệ địa phương (Yen)
- Không nói được tiếng Nhật

**Giải pháp:**

Bạn ở khách sạn, nên chỉ cần nói với lễ tân: "Áo của tôi bẩn rồi." Khách sạn sẽ:

- Tìm tiệm giặt ủi
- Thanh toán bằng tiền địa phương
- Giao tiếp bằng tiếng Nhật
- Trả lại cho bạn áo sạch

**Áp dụng vào lập trình:**

```python
# Bạn chỉ cần làm thế này
hotel.dry_clean(my_shirt)

# Không cần quan tâm chi tiết bên trong:
# - Tìm tiệm giặt
# - Chuyển đổi tiền tệ
# - Giao tiếp với nhân viên
# - Vận chuyển qua lại
```

**Lợi ích của OOP:**

- Không cần biết chi tiết implementation (cách thực hiện)
- Object tự quản lý data và functionality của mình
- Có thể làm việc với nhiều objects đồng thời
- Tin tưởng rằng mỗi object biết cách thực hiện nhiệm vụ của nó


## Dự án: OOP Coffee Machine

### Yêu cầu Project

Tạo lại Coffee Machine từ Day 15, nhưng lần này sử dụng **OOP approach**. Các chức năng giống hệt:

- In báo cáo tài nguyên
- Kiểm tra tài nguyên có đủ không
- Xử lý tiền xu
- Kiểm tra giao dịch thành công
- Pha cà phê

**Lưu ý quan trọng:**

- Không chỉnh sửa các file khác

### Classes có sẵn

**Trong main.py đã import:**

```python
from menu import Menu, MenuItem
from coffee_maker import CoffeeMaker
from money_machine import MoneyMachine
```

**4 Classes để sử dụng:**

1. **Menu** - Quản lý thực đơn đồ uống
2. **MenuItem** - Đại diện cho một món đồ uống
3. **CoffeeMaker** - Quản lý tài nguyên và pha cà phê
4. **MoneyMachine** - Xử lý thanh toán

### Coffee Machine Documentation

Documentation mô tả chi tiết từng class:

**MenuItem Class:**

- **Attributes:**
    - `name` - Tên đồ uống
    - `cost` - Giá tiền
    - `ingredients` - Nguyên liệu cần thiết

**Menu Class:**

- Methods để lấy danh sách món
- Methods để tìm món theo tên

**CoffeeMaker Class:**

- Methods kiểm tra tài nguyên
- Methods báo cáo tài nguyên
- Methods pha cà phê

**MoneyMachine Class:**

- Methods xử lý tiền xu
- Methods kiểm tra thanh toán
- Methods báo cáo lợi nhuận

### So sánh: Procedural vs OOP approach

**Procedural (Day 15):**

- Viết tất cả functions từ đầu
- Quản lý variables toàn cục
- Logic phức tạp, khó maintain

**OOP (Day 16):**

- Sử dụng pre-written classes
- Objects tự quản lý data
- Code ngắn gọn, dễ hiểu, dễ maintain
