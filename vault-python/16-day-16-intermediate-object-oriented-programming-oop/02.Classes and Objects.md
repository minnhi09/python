

## Cách Triển khai OOP (Object-Oriented Programming)

### Mô hình hóa Đối tượng (Modeling Objects)

OOP được gọi như vậy vì nó cố gắng **mô hình hóa các đối tượng trong thế giới thực**. Khi tạo một nhà hàng ảo (virtual restaurant), chúng ta cần mô hình hóa: chef, waiter, cleaner, và manager.

**Ví dụ: Mô hình hóa Waiter**

Để mô hình hóa một waiter, cần suy nghĩ về 2 điều:

**1. Nó có gì (What it has):**

- `is_holding_plate` = True/False
- `tables_responsible` =

**2. Nó làm gì (What it does):**

- `take_order()` - Nhận order và chuyển cho chef
- `take_payment()` - Nhận thanh toán và thêm tiền vào nhà hàng


### Attributes (Thuộc tính) và Methods (Phương thức)

**Attributes (Thuộc tính):**

- Là **variables** (biến) gắn liền với một object cụ thể
- Không phải là biến tự do trong code
- Là biến thuộc về một đối tượng cụ thể
- Ví dụ: `waiter.tables_responsible` - là thuộc tính của waiter, không phải biến độc lập

**Methods (Phương thức):**

- Là **functions** (hàm) mà một object cụ thể có thể thực hiện
- Không phải là hàm tự do trong code
- Là hàm gắn liền với một đối tượng
- Ví dụ: `waiter.take_order()` - là phương thức của waiter object


### Class (Lớp) và Object (Đối tượng)

**Class (Lớp):**

- Là **blueprint** (bản thiết kế) hoặc **type** (kiểu)
- Định nghĩa cấu trúc chung cho một loại đối tượng
- Quy định object đó có những thuộc tính và phương thức gì

**Object (Đối tượng):**

- Là **thực thể cụ thể** được tạo ra từ class blueprint
- Có thể tạo nhiều objects từ cùng một class
- Mỗi object có giá trị thuộc tính riêng

**Ví dụ minh họa:**

```
Class: Waiter (Blueprint)
↓
Objects tạo từ blueprint:
- Henry (waiter object)
- Betty (waiter object)
- John (waiter object)
```

Mỗi waiter object có:

- Cùng cấu trúc (attributes và methods giống nhau)
- Nhưng giá trị riêng (Henry phụ trách bàn 1-3, Betty phụ trách bàn 4-6)


### Tổng kết

**Object trong OOP là:**

- Cách kết hợp **data** (dữ liệu) và **functionality** (chức năng) trong cùng một thứ
- Có attributes (thuộc tính) - đại diện cho "những gì nó có"
- Có methods (phương thức) - đại diện cho "những gì nó làm được"

**Mối quan hệ Class - Object:**

- **Class** = Blueprint/Template (bản thiết kế)
- **Object** = Instance được tạo từ Class (phiên bản cụ thể)
- Một Class có thể tạo ra nhiều Objects khác nhau
