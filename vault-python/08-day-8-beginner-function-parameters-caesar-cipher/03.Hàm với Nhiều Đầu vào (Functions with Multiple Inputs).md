



### Mở rộng hàm với nhiều tham số

Ở bài trước, chúng ta đã học cách tạo hàm với một đầu vào. Giờ đây, ta có thể mở rộng để tạo hàm nhận nhiều đầu vào cùng lúc.

**Cú pháp thêm nhiều tham số:**

```python
def greet_with(name, location):
    print(f"Hello {name}")
    print(f"What is it like in {location}?")
```

**Cách hoạt động:**

- Các tham số (parameters) được ngăn cách bởi dấu phẩy
- Mỗi tham số có thể được sử dụng độc lập trong thân hàm
- Khi gọi hàm, cần truyền đủ số đối số (arguments) tương ứng


### Gọi hàm với nhiều đối số

**Ví dụ:**

```python
greet_with("Jack Bauer", "Nowhere")
```

**Kết quả:**

```
Hello Jack Bauer
What is it like in Nowhere?
```

Dữ liệu được truyền vào theo thứ tự: đối số thứ nhất gán cho tham số thứ nhất, đối số thứ hai gán cho tham số thứ hai.

### Positional Arguments (Đối số vị trí)

**Định nghĩa:** Là cách truyền dữ liệu vào hàm dựa trên vị trí của đối số, không chỉ định rõ tên tham số.

**Đặc điểm:**

- Thứ tự đối số phải khớp với thứ tự tham số
- Đây là cách gọi hàm mặc định trong Python
- Dễ gây lỗi nếu nhầm lẫn thứ tự

**Ví dụ minh họa vấn đề:**

```python
# Gọi đúng thứ tự
greet_with("Jack Bauer", "Nowhere")
# Output: Hello Jack Bauer
#         What is it like in Nowhere?

# Gọi SAI thứ tự
greet_with("Nowhere", "Jack Bauer")
# Output: Hello Nowhere
#         What is it like in Jack Bauer?
```

**Giải thích:** Khi đảo vị trí, `name` nhận giá trị `"Nowhere"` và `location` nhận `"Jack Bauer"`, dẫn đến kết quả sai logic.

### Ví dụ tổng quát với nhiều tham số

```python
def example(a, b, c):
    # Xử lý với a, b, c
    pass

# Trường hợp 1: Thứ tự đúng
example(1, 2, 3)
# a=1, b=2, c=3

# Trường hợp 2: Thứ tự khác
example(3, 1, 2)
# a=3, b=1, c=2
```


### Keyword Arguments (Đối số từ khóa)

**Định nghĩa:** Là cách truyền dữ liệu vào hàm bằng cách chỉ định rõ tên tham số, không phụ thuộc vào vị trí.

**Ưu điểm:**

- Rõ ràng, dễ đọc, dễ hiểu
- Không bị lỗi do nhầm thứ tự
- Code ít lỗi hơn (less error-prone)

**Nhược điểm:**

- Dòng code dài hơn
- Mất thời gian gõ nhiều hơn

**Cú pháp:**

```python
greet_with(name="Angela", location="London")
```

**Linh hoạt về thứ tự:**

```python
# Cả hai cách gọi này đều cho kết quả giống nhau
greet_with(name="Angela", location="London")
greet_with(location="London", name="Angela")
```

**Kết quả:**

```
Hello Angela
What is it like in London?
```


### So sánh hai loại đối số

**Positional Arguments:**

- Ngắn gọn, nhanh chóng
- Phụ thuộc vào thứ tự
- Phù hợp khi hàm có ít tham số và ý nghĩa rõ ràng

**Keyword Arguments:**

- Rõ ràng, an toàn
- Không phụ thuộc thứ tự
- Phù hợp khi hàm có nhiều tham số hoặc cần tránh nhầm lẫn


### Mẹo sử dụng trong thực tế

**Khi nào dùng Positional Arguments:**

- Hàm có 1-2 tham số
- Ý nghĩa tham số rõ ràng từ tên hàm
- Code ngắn gọn là ưu tiên

**Khi nào dùng Keyword Arguments:**

- Hàm có 3+ tham số
- Tham số có thể gây nhầm lẫn
- Cần code rõ ràng, dễ bảo trì


### Ghi chú quan trọng

- Luôn kiểm tra thứ tự tham số khi sử dụng positional arguments
- IDE thường gợi ý tên tham số khi gọi hàm
- Có thể tham khảo định nghĩa hàm để xem thứ tự tham số
- Sử dụng phán đoán của bạn để chọn cách gọi hàm phù hợp
